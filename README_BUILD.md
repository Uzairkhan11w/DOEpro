# Working on DOEpro

The package in `R/` is the single source of truth. The deployable single-file
app is generated from it, so the two can never drift apart.

## To change the app

1. Edit the relevant file in `R/`. Never edit `app/app.R`: it is overwritten.
2. Regenerate the single-file app and refresh the docs, then check:

```r
source("build_app.R")                       # R/ -> app/app.R (verified complete)
devtools::document()                        # roxygen -> NAMESPACE + man/
devtools::test()                            # the test suite
devtools::check()                           # full R CMD check, aim for 0/0/0
```

3. Rebuild the browser (shinylive) site and push:

```r
unlink("docs", recursive = TRUE)
shinylive::export("app", "docs")
```

```bash
git add .
git commit -m "..."
git push
```

Cloudflare Pages rebuilds https://doepro.pages.dev from `docs/` automatically,
usually within a couple of minutes. Hard-refresh (Ctrl+Shift+R) to bypass the
browser cache when checking the result.

## Layout

| Path | Purpose | Shipped to CRAN? |
|---|---|---|
| `R/` | the package source: the single source of truth | yes |
| `man/`, `NAMESPACE` | generated by `devtools::document()` | yes |
| `tests/` | the testthat suite | yes |
| `inst/CITATION` | what `citation("DOEpro")` prints | yes |
| `app/app.R` | generated by `build_app.R`; input to shinylive | no (`.Rbuildignore`) |
| `docs/` | the compiled browser app that Cloudflare serves | no (`.Rbuildignore`) |
| `build_app.R` | the generator | no (`.Rbuildignore`) |
| `DEPLOYMENT.md`, `CITATION.cff` | project documentation | no (`.Rbuildignore`) |

`build_app.R` counts the expressions it copies and refuses to write `app/app.R`
if any were lost, so a silent partial build cannot reach the deployed site.
