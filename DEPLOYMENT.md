# Maintaining DOEpro

How the project is put together and how to get a change live. For what DOEpro does and how
to cite it, see `README.md`.

---

## Making a change

The package in `R/` is the single source of truth. `app/app.R` and `docs/` are generated
from it; never edit them by hand.

```r
setwd("path/to/DOEpro")

source("build_app.R")     # R/ -> app/app.R, refuses to write an incomplete file
devtools::document()      # roxygen comments -> NAMESPACE and man/
devtools::test()          # the testthat suite
devtools::check()         # full R CMD check; keep this at 0 errors, 0 warnings, 0 notes
```

Then rebuild the browser app and push:

```r
unlink("docs", recursive = TRUE)
shinylive::export("app", "docs")
```

```bash
git add .
git commit -m "..."
git push
```

Cloudflare rebuilds https://doepro.pages.dev from `docs/` within a couple of minutes.
Hard-refresh (Ctrl+Shift+R) when checking, or the browser will serve the old cached copy.
GitHub Actions runs `R CMD check` on Windows, macOS and Linux on every push; the result is
the badge in `README.md`.

## Layout

| Path | Purpose | Goes to CRAN? |
|---|---|---|
| `R/` | package source: the single source of truth | yes |
| `man/`, `NAMESPACE` | generated by `devtools::document()` | yes |
| `tests/` | the testthat suite | yes |
| `inst/CITATION` | what `citation("DOEpro")` prints | yes |
| `app/app.R` | generated by `build_app.R`; input to shinylive | no |
| `docs/` | the compiled browser app Cloudflare serves | no |
| `build_app.R` | the generator | no |
| `DEPLOYMENT.md`, `CITATION.cff`, `.github/` | project files | no |

`.Rbuildignore` keeps the last four out of the CRAN build.

## Hosting

`shinylive` compiles the app to WebAssembly, so **R runs inside the visitor's browser**.
There is no server: no bill, no session limit, nothing to keep patched. The compiled site
is `docs/`, which is committed to the repository.

| | |
|---|---|
| **Live app** | https://doepro.pages.dev |
| **Host** | Cloudflare Pages, watching this repository |
| **Serves** | `docs/` on the `main` branch |
| **Build command** | none — `docs/` arrives already built |
| **Build output directory** | `docs` |

The same folder is also published by GitHub Pages at https://uzairkhan11w.github.io/DOEpro/.
Both work; Cloudflare is the one to quote, as it leaves the username out of the address.

**The first load takes about half a minute.** The browser must download and start a complete
copy of R, roughly 70 MB, and every plot then runs R and ggplot2 inside that sandbox. It
caches afterwards. This is inherent to running R without a server and cannot be tuned away.

If a faster copy is ever wanted, the answer is a second deployment on a real R server, not a
change to the shinylive build. `is_wasm()` already detects the browser build and hides the
PDF button there, so a server-hosted copy gains one-click PDF export for nothing. Free
options as at July 2026: **Posit Connect Cloud** (deploys from this repository; try this
first) or **shinyapps.io** (free tier, 25 active hours a month). Hugging Face Spaces is no
longer usable: Docker Spaces need a paid plan, and free Static Spaces cannot run R.

## Dependencies

Only **shiny**, **DT** and **ggplot2**, plus **rlang** and the base packages. This is
deliberate: a small, pure-R dependency list is what makes the free WebAssembly build
possible. Levene's test, the Box-Cox profile likelihood and all six post-hoc procedures are
implemented in the package rather than taken from `car` or `agricolae`, for the same reason.
`pagedown` is only suggested, and only used for PDF export where Chrome is installed.

Adding a dependency means checking it compiles to WebAssembly first, or the browser build
breaks.

## Releasing a new version

1. Raise `Version` in `DESCRIPTION`.
2. Add a heading and notes to `NEWS.md`.
3. Rebuild (`build_app.R`, `document()`, `check()`, `shinylive::export()`), commit and push.
4. Cut a release on GitHub with the tag `vX.Y.Z`.

Zenodo is linked to the repository and archives each release automatically, minting a DOI
for it. Two DOIs exist:

| DOI | Meaning |
|---|---|
| `10.5281/zenodo.21399570` | **concept DOI — cite this one.** Always resolves to the newest release. |
| `10.5281/zenodo.21399571` | version DOI, pinned to v2.0.0 |

The concept DOI never changes, so nothing needs editing when a new version goes out. It is
recorded in `README.md`, `CITATION.cff`, `inst/CITATION`, the About tab, and the footer of
every report the app produces. `CITATION.cff` drives the **Cite this repository** button;
`inst/CITATION` is what `citation("DOEpro")` prints. Keep both in step with `DESCRIPTION`.
